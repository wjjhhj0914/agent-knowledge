### 📚 2. 초기 학습 콘텐츠 컨셉 (Initial Content Concepts)

핵심 목표를 달성하기 위해, AI 에이전트가 제공할 수 있는 구체적인 학습 모듈의 컨셉입니다. 모든 콘텐츠는 **'캐릭터 기반의 스토리텔링'**을 기본으로 합니다.

#### 컨셉 1: 🗺️ 모험의 지도 (Adventure Map: Vocabulary & Storytelling)
**목표:** 새로운 어휘를 맥락 속에서 습득하고 스토리의 흐름을 따라가며 자연스럽게 익히기.

*   **콘텐츠 형태:** 캐릭터가 등장하는 짧은 인터랙티브 스토리 (5~10분 분량).
*   **학습 방식:**
    *   **"다음 단어 찾기":** 스토리 진행 중, 특정 상황에 맞는 핵심 단어(예: '빨갛다', '뛰다', '친절하다')를 그림이나 상황 제시와 함께 선택하게 합니다.
    *   **"상황 대화 완성":** 캐릭터들이 대화하는 상황에서 빈칸에 알맞은 단어를 채워 넣고, AI가 자연스러운 문장으로 완성해 줍니다.
*   **예시 시나리오:** "용감한 토끼가 숲속에서 길을 잃었어요. 토끼는 어떤 색깔의 꽃을 찾아야 할까요? (빨간색, 파란색, 노란색 중 선택)"

#### 컨셉 2: 🎨 감정의 색칠 놀이 (Emotion Coloring: Grammar & Expression)
**목표:** 기초 문장 구조를 익히고, 자신의 감정을 언어로 표현하는 연습을 통해 문법적 이해를 높이기.

*   **콘텐츠 형태:** 감정 기반의 시각적 활동 및 대화 연습.
*   **학습 방식:**
    *   **"내 기분 표현하기":** AI가 다양한 감정(기쁨, 슬픔, 화남, 신남)을 표현하는 캐릭터를 제시하고, 아이가 그 감정에 맞는 문장이나 표현을 선택하여 대화하게 합니다.
    *   **"상황별 문장 만들기":** "나는 오늘 기분이 좋아. 왜 그럴까?"와 같은 간단한 질문에 대해 아이가 문장을 완성하면, AI가 문법적으로 정확하게 교정하고 칭찬해 줍니다.
*   **예시 시나리오:** "친구에게 선물을 받았을 때 기분은 어때? (기쁘다/슬프다). 네가 기쁠 때 하는 말은?"

#### 컨셉 3: 🧩 퍼즐 보물찾기 (Puzzle Treasure Hunt: Vocabulary Reinforcement)
**목표:** 학습한 어휘를 장기 기억으로 전환하고, 게임 요소를 통해 반복 학습을 유도하기.

*   **콘텐츠 형태:** 단어 카드, 숨겨진 그림, 미션 기반의 퀴즈 게임.
*   **학습 방식:**
    *   **"보물 찾기 미션":** 특정 단어(예: '사과')를 찾기 위해 그림 속에서 해당 사과를 찾아내거나, 사과와 관련된 단어를 맞추는 미션을 수행합니다.
    *   **"단어 조합 퍼즐":** 제시된 단어 조각들을 조합하여 새로운 문장이나 스토리를 만들어보게 하여, 어휘의 관계를 이해하게 합니다.
*   **예시 시나리오:** "보물 상자를 열려면 '노란색'과 '작다'라는 단어를 찾아야 해. 그림 속에서 이 단어를 찾아봐!"

## 🚀 AI Agent 핵심 기능 기술 아키텍처 설계 (High-Level Blueprint)

CEO의 지시에 따라, 초등학교 저학년 대상 언어 학습 AI 에이전트의 핵심 기능(스토리텔링 기반 인터랙티브 학습, 개인화 피드백)을 구현하기 위한 초기 기술 아키텍처의 큰 그림을 설계합니다.

이 아키텍처는 **'몰입감 있는 경험 제공'**과 **'학습 데이터 기반의 개인화'**라는 두 가지 목표를 달성하는 데 중점을 둡니다.

---

### 1. AI 모델 선정 전략 (AI Model Selection Strategy)

학습 경험의 질을 높이고, 친근한 톤앤매너를 유지하며, 복잡한 언어 이해 및 생성 능력을 확보하기 위해 **멀티모달(Multi-modal) 접근**을 권장합니다.

| 기능 영역 | 추천 AI 모델 유형 | 선정 이유 |
| :--- | :--- | :--- |
| **콘텐츠 생성 (Storytelling & Dialogue)** | **GPT-4o / Claude 3.5 Sonnet** | 높은 맥락 이해 능력, 창의적인 스토리텔링 및 대화 생성 능력으로 '모험의 지도' 콘텐츠의 질을 극대화합니다. 친근하고 재미있는 톤앤매너 구현에 최적화됩니다. |
| **학습 데이터 분석 및 피드백 (NLU/Assessment)** | **Fine-tuned LLM / Custom RAG** | 학생의 답변(선택, 문장 생성)을 분석하여 학습 수준(어휘, 문법)을 정확히 진단하고, 즉각적이고 맞춤화된 피드백을 생성하는 데 사용합니다. |
| **감정 및 상호작용 (Engagement Layer)** | **Small, Specialized NLP Models** | 실시간으로 학생의 감정 상태(흥미, 좌절, 자신감)를 감지하여, 다음 스토리 분기점이나 난이도를 조절하는 인터랙션 레이어에 사용됩니다. |
| **멀티모달 통합 (Visual Input)** | **Vision API (선택적)** | 추후 이미지 기반 학습 모듈(예: 그림 보고 단어 맞추기)을 추가할 경우, 이미지 인식 기능을 통합하여 학습 경험을 시각적으로 풍부하게 합니다. |

**🔑 핵심 모델 결정:** 초기 MVP 단계에서는 **GPT-4o (또는 동급의 최신 LLM)**를 중심으로 구축하여, 스토리 생성과 맞춤형 대화라는 핵심 경험을 확보하고, 이후 데이터 파이프라인을 통해 Fine-tuning을 진행하는 것을 추천합니다.

---

### 2. 데이터 파이프라인 구상 (Data Pipeline Design)

학습 데이터의 수집, 처리, 활용을 위한 안정적인 파이프라인을 설계합니다. 이 파이프라인은 **'실시간 피드백 루프'**를 중심으로 구축되어야 합니다.

#### A. 데이터 수집 단계 (Ingestion Layer)

1.  **사용자 입력 (Interaction Log):** 학생이 AI와 상호작용한 모든 대화 기록, 선택지, 생성된 문장 등 원시 데이터 수집.
2.  **학습 목표 데이터 (Curriculum Data):** 사전에 정의된 어휘 목록, 문법 규칙, 스토리 분기점 등의 마스터 데이터베이스.
3.  **사용자 프로필 데이터 (Profile Data):** 학생의 초기 레벨, 선호 주제, 학습 속도 등 초기 설정 데이터.

#### B. 데이터 처리 및 저장 단계 (Processing & Storage Layer)

1.  **NLP 분석 모듈:** 수집된 사용자 입력을 분석하여 **정답 여부, 문법적 오류, 감정 상태**를 추출합니다.
2.  **Vector Database (RAG):** 모든 학습 콘텐츠(스토리, 어휘 정의)를 벡터 임베딩하여 저장합니다. 이는 AI가 맥락에 맞는 정보를 즉시 검색(Retrieval)하고 생성(Generation)하는 데 사용됩니다.
3.  **Relational Database (SQL/NoSQL):** 학생의 누적 학습 진도, 숙련도 점수, 목표 달성 상태 등 정량적 데이터를 저장하여 대시보드(학부모용) 구축의 기반을 마련합니다.

#### C. 학습 및 피드백 단계 (Learning & Feedback Loop)

1.  **Personalization Engine:** 저장된 학습 데이터(Relational DB)와 현재 입력(NLP 분석)을 기반으로, **다음 단계에서 가장 적합한 스토리 분기점이나 학습 목표**를 결정합니다.
2.  **Feedback Generation:** Personalization Engine의 결정에 따라, LLM이 **'친근하고 칭찬하는 톤'**으로 맞춤형 피드백과 다음 학습 콘텐츠를 실시간으로 생성하여 사용자에게 제공합니다.

### 3. 아키텍처 요약 (System Architecture Summary)

# 🧠 Vision Framework 기반 시선 추적 및 표정 분석 로직 기술 설계도 (Python/Markdown)

## 🎯 목표

## ⚙️ 아키텍처 개요 (High-Level Flow)
시스템은 크게 세 단계로 구성됩니다: **입력 처리 (Input Processing) → 시각 특징 추출 (Feature Extraction) → 감정 및 시선 추론 (Inference & Mapping)**.

```mermaid

    G --> H[AI Agent Feedback Generator];
    H --> I[학습 인터페이스 (UI/UX)];
```

## 🛠️ 상세 모듈 설계 (Module Details)

### 1. 입력 처리 모듈 (Input Processing Module)
**목표:** 비디오 프레임에서 얼굴 영역을 정확히 분리하고 핵심 랜드마크를 추출합니다.
**기술 스택:** OpenCV, MediaPipe Face Mesh

| 단계 | 기능 설명 | 주요 기술 | 출력 데이터 |
| :--- | :--- | :--- | :--- |
| **1. 프레임 로딩** | 입력된 비디오 스트림에서 프레임을 분리. | OpenCV | Raw Frame |
| **2. 얼굴 감지** | 프레임 내에서 얼굴 영역을 탐지하고 바운딩 박스(Bounding Box)를 생성. | MediaPipe Face Detection | Face Bounding Box |
| **3. 랜드마크 추출** | 얼굴 내의 주요 랜드마크(눈, 코, 입, 눈썹) 좌표를 추출. | MediaPipe Face Mesh | 3D Landmark Coordinates |

### 2. 시선 추적 모듈 (Gaze Estimation Module)
**목표:** 사용자의 시선이 화면 내의 특정 학습 요소(예: 색칠 도안의 특정 영역)에 얼마나 집중하고 있는지를 측정합니다.
**기술 스택:** 3D Landmark 좌표, 시선 벡터 계산

**핵심 로직:**
1.  **시선 벡터 계산:** 눈의 중심점(Pupil/Eye Center)과 랜드마크를 사용하여 카메라 중심(Center of Frame)과의 상대적인 벡터를 계산합니다.
2.  **집중 영역 정의:** 특정 학습 객체(예: '기쁨' 감정 도안)의 경계 상자를 정의하고, 사용자의 시선 벡터가 이 경계 상자 내에 얼마나 머무르는지 계산합니다.
3.  **몰입도 점수 산출:** 시선이 목표 영역 내에 머무르는 시간 및 각도를 기반으로 **몰입도 점수 ($\text{GazeScore}$)**를 산출합니다.

$$\text{GazeScore} = \text{CosineSimilarity}(\vec{V}_{\text{Gaze}}, \vec{V}_{\text{Target}}) \times \text{Proximity}(\text{EyeCenter}, \text{TargetArea})$$

### 3. 표정 분석 모듈 (Facial Expression Recognition Module)
**목표:** 추출된 얼굴 이미지에서 감정 상태를 분류합니다.
**기술 스택:** Pre-trained CNN (예: FER-2013 기반 Fine-tuning, 또는 Vision Transformer)

**핵심 로직:**
1.  **이미지 전처리:** 얼굴 영역만 잘라내어 표준화된 크기로 조정합니다.
2.  **특징 추출:** CNN 모델을 통해 이미지의 픽셀 특징을 추출합니다.
3.  **감정 분류:** 추출된 특징을 기반으로 미리 정의된 감정 클래스(기쁨, 슬픔, 화남, 중립 등) 중 가장 확률이 높은 감정 레이블을 출력합니다.
4.  **감정 강도 산출:** 각 감정 클래스에 대한 확률 분포를 통해 해당 감정의 **강도 ($\text{EmotionIntensity}$)**를 산출합니다.

### 4. 맥락 매핑 엔진 (Contextual Mapping Engine)
**목표:** 추론된 시선 및 표정 데이터를 학습 목표와 연결하여 의미 있는 피드백을 생성합니다.
**기술 스택:** LLM (Large Language Model) 기반 추론

**로직:**
1.  **입력 데이터:** $\text{GazeScore}$, $\text{EmotionIntensity}$, 현재 학습 모듈 정보 (예: '감정의 색칠 놀이', 목표 색상: 노랑).
2.  **규칙 기반 매핑:**
    *   만약 $\text{EmotionIntensity}$가 '기쁨'이고 $\text{GazeScore}$가 목표 색칠 영역에 80% 이상이라면, **긍정적 피드백**을 생성합니다.
    *   만약 $\text{EmotionIntensity}$가 '슬픔'이고 $\text{GazeScore}$가 목표 영역을 벗어난다면, **교정적 피드백**을 생성합니다.
3.  **LLM 프롬프트 구성:** 이 매핑 결과를 바탕으로 AI 에이전트에게 구체적인 학습 조언(예: "와! 노란색을 정말 멋지게 채웠구나! 기쁨의 색깔이 느껴져!")을 생성하도록 지시합니다.

$$\text{Feedback} = \text{LLM}(\text{Context}, \text{GazeScore}, \text{EmotionIntensity}, \text{LearningGoal})$$

# 🗺️ AI Agent 학습 지원 웹페이지 UI/UX 설계 (Play & Explore Theme)

## 1. 디자인 시스템 정의 (Visual System)

브랜드 정체성인 '놀이'와 '탐험' 감성을 극대화하기 위해 감정(Emotion)과 색상(Color)을 직접 연결하는 팔레트를 정의합니다.

### 1.1. 컬러 팔레트 (Color Palette)
| 역할 | 색상 이름 | HEX 코드 | 용도 및 감성 |
| :--- | :--- | :--- | :--- |
| **Primary (탐험)** | Deep Ocean Blue | `#1E40AF` | 신뢰감, 깊이 있는 탐험, 안정감 |
| **Secondary (놀이)** | Sunny Yellow | `#FFD700` | 활력, 긍정, 즐거움 (강조 색상) |
| **Accent (감정)** | Coral Pink | `#FF6B6B` | 감정 표현, 따뜻함, 상호작용 유도 |
| **Neutral (배경)** | Soft Cream | `#FFFBF5` | 부드러운 배경, 학습 공간 제공 |
| **Tertiary (보물)** | Forest Green | `#4CAF50` | 성장, 자연, 발견 (보상) |

### 1.2. 타이포그래피 (Typography)
*   **Font Family:** Pretendard (친근하고 가독성이 높은 고딕 계열)
*   **Heading (H1, H2):** Poppins (역동성과 명료성 강조)
*   **Body Text:** Pretendard (최대 가독성 확보)
*   **스타일:** 모든 텍스트에 그림자(Soft Shadow)를 적용하여 입체감과 친근함을 부여합니다.

### 1.3. 아이콘 및 요소 스타일
*   **아이콘 스타일:** Line/Outline 기반, 모서리가 둥글게 처리된(Rounded Corners) 친근한 스타일.
*   **상호작용:** 모든 버튼과 카드에는 마우스를 올렸을 때 색상이 부드럽게 변화(Hover Effect)하며, 클릭 시 미세한 애니메이션(Bounce/Pulse)을 추가하여 '놀이' 요소를 강화합니다.
